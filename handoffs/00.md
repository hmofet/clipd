# Handoff: Network Clipboard Web App ("clipd") — Railway Edition

## Objective

Build and deploy a multi-tab clipboard web app in **Go** on **Railway**. Users open it in a browser, create named tabs, and paste text. All tabs are shared — what you paste on one device appears on every other device that loads the page.

## Application Requirements

### Core Features

1. **Multi-tab clipboard**
   - Create, rename, delete tabs
   - Each tab holds a single block of text (arbitrary length)
   - Tab content persists across app restarts (PostgreSQL)
   - Tabs have a user-defined name (default: "Tab 1", "Tab 2", etc.)
   - Tab order is preserved

2. **On page load**
   - All existing tabs are loaded from the backend
   - The first tab is selected by default
   - Tab content is displayed immediately

3. **Markdown toggle**
   - Per-tab toggle switch: raw text ↔ rendered Markdown
   - Toggle state is NOT persisted — always defaults to raw on load
   - When Markdown is on: rendered HTML shown below the editor (or replaces it)
   - When Markdown is off: plain textarea only

4. **Dark mode**
   - Toggle switch in the header/toolbar area
   - Respects `prefers-color-scheme` on first load (auto-detect system preference)
   - User's choice is saved to `localStorage` and restored on next visit
   - Two coherent themes: light and dark. Use CSS custom properties (`--bg`, `--fg`, `--border`, `--accent`, etc.) so the entire theme switches via a single class on `<body>`
   - Dark theme: dark background (#1a1a2e or similar), light text, muted borders
   - Light theme: white/near-white background, dark text
   - The textarea, tabs, and all UI elements must respect the active theme

5. **Context menu for tab management**
   - Right-click on a tab → custom context menu with "Rename" and "Delete" options
   - NO visible rename/delete buttons in the UI — context menu is the only way
   - "Rename" opens an inline edit (turn the tab name into an input field, press Enter or blur to save)
   - "Delete" shows a confirmation prompt before deleting
   - The [+ New Tab] button remains as a visible button (this is not in the context menu)
   - Left-click on a tab selects it (as before)
   - The context menu closes when clicking elsewhere
   - Prevent the browser's default context menu only on tab elements

6. **Auto-save**
   - Content saves automatically on a short debounce (e.g., 500ms after last keystroke)
   - Visual indicator: small "Saved ✓" or "Saving..." text
   - No explicit save button needed

7. **Favicon**
   - Embed a small SVG favicon inline in the HTML (no external file)
   - Design: simple clipboard icon — a rectangle with a small clip/tab at the top
   - Use a `<link rel="icon" href="data:image/svg+xml,...">` approach so it's self-contained in the single HTML file
   - Should look clean at 16x16 and 32x32

### API Design (REST)

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/tabs` | List all tabs (id, name, order) |
| `GET` | `/api/tabs/{id}` | Get tab content |
| `POST` | `/api/tabs` | Create new tab (returns new tab object) |
| `PUT` | `/api/tabs/{id}` | Update tab content and/or name |
| `DELETE` | `/api/tabs/{id}` | Delete tab |
| `PUT` | `/api/tabs/reorder` | Reorder tabs (accepts ordered list of IDs) |

Tab IDs: use a simple auto-incrementing integer (PostgreSQL `SERIAL`).

### Database Schema (PostgreSQL)

Single table:

```sql
CREATE TABLE IF NOT EXISTS tabs (
    id          SERIAL PRIMARY KEY,
    name        TEXT NOT NULL DEFAULT 'New Tab',
    content     TEXT NOT NULL DEFAULT '',
    tab_order   INTEGER NOT NULL DEFAULT 0,
    created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

- The app runs `CREATE TABLE IF NOT EXISTS` on startup — no migration tool needed
- `tab_order` is an integer used for sorting; reorder endpoint updates these values
- `updated_at` is set by the Go code on every write, not a DB trigger

```
│  clipd                          [☀/🌙]              │
├──────────────────────────────────────────────────────┤

│   │                                              │   │
│   └──────────────────────────────────────────────┘   │
│                                          Saved ✓     │
└──────────────────────────────────────────────────────┘
```

### Context Menu (on right-click of a tab)

```
┌─────────────┐
│  ✏ Rename   │
│  🗑 Delete   │
└─────────────┘
```

- Positioned at the mouse cursor
- Styled consistently with the active theme (light/dark)
- Dismisses on click outside or Escape key

### Styling

- Monospace font for the textarea (`font-family: 'Courier New', monospace`)
- CSS custom properties for theming — define all colors as variables
- Dark mode toggle in header, state persisted in `localStorage`
- No CSS framework — just a `<style>` block in the embedded HTML
- Responsive enough to work on a phone browser but desktop is the primary target
- Clean, utility-first aesthetic — this is a tool, not a product

### Markdown Rendering

- Load `marked.js` from CDN (`https://cdn.jsdelivr.net/npm/marked/marked.min.js`)
- When toggle is ON: hide textarea, show rendered HTML in a `<div>` with basic prose styling
- When toggle is OFF: show textarea, hide rendered div
- Switching back to OFF should preserve the raw text exactly as typed
- Markdown rendered view must also respect dark/light theme

## Implementation Notes for Worker

### Go-specific guidance (Rocky is learning Go)

- Write **idiomatic Go** — this is a learning project. Use proper error handling (`if err != nil`), named return values where clear, and standard project layout.
- Keep it simple. No gorilla/mux — use `http.ServeMux` from stdlib (Go 1.22+ supports method routing: `mux.HandleFunc("GET /api/tabs", handler)`).
- Use `embed.FS` to embed `index.html` into the binary.
- Use `encoding/json` for JSON marshaling.
- Use `os.Getenv` for configuration (`PORT`, `DATABASE_URL`) — no flag package needed for Railway.
- Add brief comments explaining Go patterns for Rocky's benefit.

### PostgreSQL / pgx notes

- Use `github.com/jackc/pgx/v5/pgxpool` for connection pooling.
- Open pool once at startup using `DATABASE_URL` from environment.
- Run `CREATE TABLE IF NOT EXISTS` on startup to bootstrap the schema.
- Use parameterized queries everywhere (`$1`, `$2`, etc.) — never string-concatenate user input into SQL.
- Use `context.Background()` or request-scoped contexts for queries.
- Close the pool on shutdown with `defer pool.Close()`.

Example connection setup:
```go
pool, err := pgxpool.New(context.Background(), os.Getenv("DATABASE_URL"))
if err != nil {
    log.Fatalf("Unable to connect to database: %v", err)
}
defer pool.Close()
```

### Error handling

- Return proper HTTP status codes (400, 404, 500).
- Log errors to stdout (Railway captures stdout as logs).

### Frontend notes

- **Dark mode**: Use a `<body class="dark">` or `<body class="light">` approach. Define all colors as CSS custom properties on `body.light` and `body.dark`. On load, check `localStorage.getItem('theme')` first; if absent, check `window.matchMedia('(prefers-color-scheme: dark)')`. Toggle button switches the class and saves to `localStorage`.

- **Context menu**: Create a hidden `<div>` for the menu. On `contextmenu` event on a tab element, prevent default, position the div at `e.clientX`/`e.clientY`, and show it. Each menu item triggers its action and hides the menu. Click outside or `Escape` hides it. Store the target tab ID when opening the menu so rename/delete know which tab to act on.

- **Favicon**: Use an inline SVG data URI in a `<link>` tag. Keep the SVG minimal — a rounded rectangle with a small clip shape at top center. Use a single accent color.

## Railway-Specific Configuration

### How Railway builds Go apps

Railway auto-detects Go projects by the presence of `go.mod`. It runs `go build` and executes the resulting binary. No `Procfile`, `Dockerfile`, or `railway.toml` needed for a standard Go project.

If the auto-detection fails or needs customization, create a `Procfile`:
```
web: ./clipd
```

But this should not be necessary.

### Environment variables

| Variable | Source | Notes |
|----------|--------|-------|
| `PORT` | Railway (automatic) | Railway assigns a port; the app must listen on it |
| `DATABASE_URL` | Railway (automatic, from Postgres add-on) | Full connection string including credentials |

The app must read both from `os.Getenv`. No hardcoded values.

### Logs

View logs in the Railway dashboard under the clipd service → **Deployments** → click a deployment → **Logs**. Or use the Railway CLI:

```bash
railway logs
```

## What NOT To Do

- **No authentication** — this is a personal tool, not multi-tenant
- **No HTTPS configuration** — Railway handles TLS termination automatically
- **No Dockerfile** — Railway's Go buildpack handles it
- **No database migrations tool** — a single `CREATE TABLE IF NOT EXISTS` on startup is sufficient
- **No frontend build step** — single HTML file, no npm/webpack
- **No CSS framework** — inline `<style>` block with CSS custom properties for theming
- **No visible rename/delete buttons** — context menu only
- **Don't hardcode the port or database URL** — read from environment variables
