# Handoff: clipd Magic Link Authentication

## Objective

Add per-user tab isolation to clipd using magic link (email) authentication. No passwords. User enters their email, clicks a link in the email, and gets a session. Each user sees only their own tabs.

---

## How It Works

```
1. User visits clipd → sees login screen (email input)
2. User enters email → server generates a random token, stores it, sends email with link
3. User clicks link → server validates token, creates session, sets cookie
4. User is now logged in → sees only their own tabs
5. Session lasts 30 days (refreshed on each visit)
6. "Log out" button clears the session
```

---

## Email Provider: Resend

**Why Resend**: Simple REST API, no SDK needed (just an HTTP POST from Go), free tier covers this use case.

**Free tier**: 3,000 emails/month, 100/day. For a personal clipboard tool this is more than enough.

**Setup**:

1. Sign up at [resend.com](https://resend.com)
2. Add and verify a sending domain (or use Resend's onboarding domain for testing)
3. Generate an API key
4. Add the API key as a Railway environment variable: `RESEND_API_KEY`

**Sending an email from Go** (no SDK, just net/http):

```go
func sendMagicLink(to, token string) error {
    link := fmt.Sprintf("%s/auth/verify?token=%s", os.Getenv("BASE_URL"), token)

    body := fmt.Sprintf(`{
        "from": "clipd <noreply@%s>",
        "to": ["%s"],
        "subject": "Your clipd login link",
        "html": "<p>Click to log in:</p><p><a href=\"%s\">Open clipd</a></p><p>This link expires in 15 minutes.</p>"
    }`, os.Getenv("RESEND_DOMAIN"), to, link)

    req, _ := http.NewRequest("POST", "https://api.resend.com/emails", strings.NewReader(body))
    req.Header.Set("Authorization", "Bearer "+os.Getenv("RESEND_API_KEY"))
    req.Header.Set("Content-Type", "application/json")

    resp, err := http.DefaultClient.Do(req)
    if err != nil {
        return err
    }
    defer resp.Body.Close()

    if resp.StatusCode != 200 {
        return fmt.Errorf("resend API returned %d", resp.StatusCode)
    }
    return nil
}
```

## Database Schema Changes

Add two new tables. Keep the existing `tabs` table but add a `user_id` column.

```sql
-- Users table: one row per unique email
CREATE TABLE IF NOT EXISTS users (
    id         SERIAL PRIMARY KEY,
    email      TEXT NOT NULL UNIQUE,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Magic link tokens: short-lived, one-time use
CREATE TABLE IF NOT EXISTS auth_tokens (
    id         SERIAL PRIMARY KEY,
    email      TEXT NOT NULL,
    token      TEXT NOT NULL UNIQUE,
    expires_at TIMESTAMPTZ NOT NULL,
    used       BOOLEAN NOT NULL DEFAULT FALSE,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Sessions: long-lived, tied to a user
CREATE TABLE IF NOT EXISTS sessions (
    id         SERIAL PRIMARY KEY,
    user_id    INTEGER NOT NULL REFERENCES users(id),
    token      TEXT NOT NULL UNIQUE,
    expires_at TIMESTAMPTZ NOT NULL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Add user_id to existing tabs table
ALTER TABLE tabs ADD COLUMN IF NOT EXISTS user_id INTEGER REFERENCES users(id);
```

**Migration note**: The `ALTER TABLE` handles the transition from the current shared-tabs design. Existing tabs (with `user_id = NULL`) will be orphaned — this is fine. They can be cleaned up manually or left alone.

## New API Endpoints

| Method | Endpoint | Auth? | Description |
|--------|----------|-------|-------------|
| `POST` | `/auth/login` | No | Accept `{"email": "..."}`, generate token, send magic link email |
| `GET` | `/auth/verify?token=xxx` | No | Validate token, create session, set cookie, redirect to `/` |
| `POST` | `/auth/logout` | Yes | Clear session cookie and delete session from DB |
| `GET` | `/auth/me` | Yes | Return current user's email (for UI display) |

All existing `/api/tabs/*` endpoints now require a valid session cookie. They operate only on tabs belonging to the authenticated user.

### Auth Flow Detail

**`POST /auth/login`**:
1. Receive `{"email": "user@example.com"}`
2. Normalize email: `strings.TrimSpace(strings.ToLower(email))`
3. Generate a random token: 32 bytes, hex-encoded (64 chars)
4. Insert into `auth_tokens` with `expires_at = NOW() + 15 minutes`
5. Send email via Resend API with the magic link
6. Return `{"ok": true, "message": "Check your email"}` — always return success even if email doesn't exist (prevent enumeration)

**`GET /auth/verify?token=xxx`**:
1. Look up token in `auth_tokens` where `used = FALSE` and `expires_at > NOW()`
2. If not found or expired → show error page ("Link expired or invalid")
3. Mark token as `used = TRUE`
4. Upsert user in `users` table (insert if email doesn't exist, get ID if it does)
5. Generate a session token: 32 bytes, hex-encoded
6. Insert into `sessions` with `expires_at = NOW() + 30 days`
7. Set cookie: `session=<token>; Path=/; HttpOnly; SameSite=Lax; Secure; Max-Age=2592000`
8. Redirect to `/`

**Session middleware**:
- Every request to `/api/*` checks for the `session` cookie
- Look up session in DB where `expires_at > NOW()`
- If valid → attach `user_id` to the request context
- If missing/invalid → return `401 Unauthorized`
- On each valid request, optionally refresh `expires_at` to extend the session (rolling window)

## Changes to Existing Endpoints

All `/api/tabs*` handlers must:

1. Extract `user_id` from the request context (set by auth middleware)
2. Add `WHERE user_id = $N` to all SELECT queries
3. Set `user_id` on INSERT
4. Include `AND user_id = $N` on UPDATE and DELETE (prevents accessing other users' tabs)

Example — the `GET /api/tabs` handler changes from:

```go
rows, err := pool.Query(ctx, "SELECT id, name, tab_order FROM tabs ORDER BY tab_order")
```

to:

```go
rows, err := pool.Query(ctx, "SELECT id, name, tab_order FROM tabs WHERE user_id = $1 ORDER BY tab_order", userID)
```

## Security Notes

- **Tokens are one-time use**: Once a magic link is clicked, the token is marked `used = TRUE` and cannot be reused
- **Tokens expire in 15 minutes**: Short-lived to limit the window of interception
- **Session cookies are HttpOnly**: JavaScript cannot read them — protects against XSS
- **Session cookies are Secure**: Only sent over HTTPS (Railway provides TLS)
- **SameSite=Lax**: Prevents CSRF for state-changing requests
- **Email enumeration prevented**: `/auth/login` always returns success regardless of whether the email exists
- **Rate limiting**: Not implemented in this version. For a personal tool this is fine. If exposed publicly, add rate limiting on `/auth/login` (e.g., max 5 requests per email per hour)
- **Token generation**: Use `crypto/rand` for all tokens — never `math/rand`

```go
import "crypto/rand"

func generateToken() (string, error) {
    b := make([]byte, 32)
    _, err := rand.Read(b)
    if err != nil {
        return "", err
    }
    return hex.EncodeToString(b), nil
}
```

## What NOT To Do

- **No passwords** — magic link only
- **No OAuth / Google sign-in** — email is simpler and universal
- **No email verification step** — the magic link IS the verification
- **No user profile / settings page** — just email and tabs
- **No "forgot password" flow** — there is no password to forget
- **No invitation system** — anyone with the URL can sign up by entering their email
- **No admin panel** — all users are equal
- **No rate limiting in v1** — add later if needed

## Success Criteria

1. Unauthenticated users see a login screen with an email input
2. Submitting an email sends a magic link via Resend
3. Clicking the magic link logs the user in and redirects to the tab UI
4. Each user sees only their own tabs
5. Creating/editing/deleting tabs affects only the logged-in user's data
6. "Log out" clears the session and returns to the login screen
7. Sessions persist for 30 days (user doesn't need to re-authenticate daily)
8. Dark mode and all existing features continue to work
9. The login screen respects the dark/light theme
